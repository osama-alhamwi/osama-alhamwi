## 🚀 What I do

I integrate AI into real products rather than demos. My work sits at the intersection of
three things:

- **🤖 AI integration** — wiring LLM and vision APIs into systems people use daily.
- **🌐 Web engineering** — from a single self-contained HTML file to full-stack apps.
- **🎯 Tech for good** — education platforms, non-profit management, community tools.

I care about software that keeps working when the network is slow, the budget is zero, and
the user isn't a developer. That constraint shapes how I build: offline-first where possible,
no dependency I can't justify, and documentation someone else can actually follow.

## 🌱 Featured — Namaa: Self-Growth Program

A complete personal-development system in **one HTML file**. No account, no server, no build
step — every byte of user data stays in the browser. Bilingual (EN / AR with full RTL),
light/dark themed, and religion-neutral by design so it fits any community.

> **87 tasks · 8 life dimensions · 12 modules · 0 dependencies**

Behind the simple surface: a config-driven task engine, isolated feature modules that each own
their storage namespace, and a proactive companion that reads your rhythm on-device to surface
one timely action instead of a wall of notifications.

## 🧭 How I think about building

- **Constraints are the design.** Zero budget and weak connectivity aren't obstacles to work
  around — they decide the architecture.
- **Documentation is part of the deliverable.** Code nobody can run is code nobody will use.
- **Be honest about trade-offs.** Every project I ship names its own weak points; reviewers
  find them anyway, and owning them first is cheaper than defending them later.
